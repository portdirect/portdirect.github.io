Openstack by the OpenStack Foundation

The industry standard open source platform for cloud computing. Originally founded in 2010 by Rackspace and NASA as a collection of software projects to develop and manage cloud infrastructure in a data center, it enables Infrastructure as a service (IAAS). OpenStack can provide an open and flexible vendor agnostic framework that many large enterprises have adopted as the underlying technology to drive transformation to software-defined infrastructure and support their business operations. Users cite significant competitive advantage from OpenStack with dramatically reduced provisioning times from weeks or months to minutes, and freedom from being locked in to proprietary systems.

Kubernetes by Google

An open source container cluster manager initially developed by Google, benefitting from their experience of running containers at massive scale. The project is owned and powered by a diverse set of contributors, and is part of the Linux Foundations Cloud Native Computing Foundation. Kubernetes is an open source orchestration system for containers, handling scheduling of applications onto nodes in a compute cluster and actively managing workloads to ensure that their state matches the users’ desires. It provides a simple and resilient “platform for automating deployment, scaling, and operations of application containers across clusters of hosts”.

Mesos by the Apache Software Foundation

Built at UC Berkeley. Mesos was designed for operations at very large scale. Tested and refined by Twitter, Apple’s Siri, and Airbnb, and now supporting some of the largest applications in the world, Mesos supports different types of workloads to build truly modern applications. These distributed workloads include container orchestration (Mesos containers, Docker, and K8s), analytics (Spark), big-data technologies (Kafka, Cassandra) and much more.

Docker Swarm by Docker INC

Native clustering for Docker. It turns a pool of Docker hosts into a single, virtual Docker host. Because Docker Swarm serves the standard Docker API, any tool that already communicates with a Docker daemon can use Swarm to transparently scale to multiple hosts.

Ansible by Red Hat

A radically simple IT automation engine that automates cloud provisioning, configuration management, application deployment, intra-service orchestration, and many other IT needs. Being designed for multi-tier deployments since day one, modeling your IT infrastructure by describing how all of your systems inter-relate, rather than just managing one system at a time. Without agents or custom security infrastructure, it's easy to deploy using a very simple language that allow you to describe your automation jobs in a way that approaches plain English.

Puppet by Puppet Labs

Puppet provides a standard way of delivering and operating software, no matter where it runs. With the Puppet approach, you define what you want your apps and infrastructure to look like using a common easy-to-read language. From there you can share, test and enforce the changes you want to make across your datacenter. And at every step of the way, you have the visibility and reporting you need to make decisions and prove compliance. The result: you get a standard way of automating all of it, at scale.

Docker Compose by Docker INC

Distributed applications consist of many small applications that work together. Docker transforms these applications into individual containers that are linked together. Instead of having to build, run and manage each individual container, Docker Compose allows you to define your multi-container application with all of its dependencies in a single file, then spin your application up in a single command. Your application’s structure and configuration are held in a single place, which makes spinning up applications simple and repeatable everywhere.

Salt Open by SaltStack

Salt Open is created and maintaind by SaltStack for complex systems management at scale. From one to tens of thousands of servers, and fast enough to communicate with them in seconds. Salt is a new approach to infrastructure management built on a dynamic communication bus. Salt can be used for data-driven orchestration, remote execution for any infrastructure, configuration management for any application stack, and much more.
